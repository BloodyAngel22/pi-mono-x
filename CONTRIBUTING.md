# Участие в разработке pi-mono-x

Принимаются любые contributions. Вы можете участвовать через:

- **Issues** — баг-репорты, предложения новых фич, идеи, вопросы.
- **Forks** — форкните репозиторий, внесите изменения и откройте PR.

## Перед отправкой PR

```bash
npm run check
./test.sh
```

Обе команды должны проходить без ошибок.

Если вы добавляете нового провайдера в `packages/ai`, смотрите требования к тестам в `AGENTS.md`.

## Философия

pi-mono-x стремится быть на уровне [claude-code](https://github.com/anthropics/claude-code) или лучше. Минимализм оригинального pi не был самоцелью — он мешал комфортной работе. Фичи, повышающие удобство и продуктивность, приветствуются.
