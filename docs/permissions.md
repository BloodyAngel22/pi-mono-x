# Настройка прав доступа (Permissions)

Система прав контролирует, какие действия агент выполняет автоматически, а какие требуют подтверждения.

## Файлы конфигурации

| Файл | Область |
|------|---------|
| `~/.pi/agent/permissions.json` | Глобальные (все проекты) |
| `<проект>/.pi/permissions.json` | Локальные (текущий проект, приоритет выше) |

## Формат

```json
{
  "defaultPolicy": "ask",
  "bash": {
    "allow": ["паттерн1", "паттерн2"],
    "ask": ["паттерн3"],
    "deny": ["паттерн4"]
  },
  "mcp": {
    "allow": ["имя_инструмента"],
    "deny": ["имя_инструмента"]
  },
  "file": {
    "allow": ["паттерн_пути"],
    "deny": ["паттерн_пути"]
  }
}
```

## Политики

| Политика | Описание |
|----------|----------|
| `allow` | Выполнять без подтверждения |
| `ask` | Спрашивать каждый раз |
| `deny` | Блокировать полностью |

`defaultPolicy` -- политика по умолчанию для всего, что не попало под правила. Рекомендуется `"ask"`.

## Секции

### bash -- команды терминала

Паттерны с `*` (glob). Для составных команд (`&&`, `||`, `;`, `|`) побеждает самая строгая политика.

```json
{
  "bash": {
    "allow": [
      "cat *", "ls", "ls *", "pwd",
      "echo *", "which *",
      "find *", "grep *", "rg *", "fd *",
      "head *", "tail *", "wc *",
      "sort *", "uniq *", "diff *"
    ],
    "ask": [
      "rm *", "git *",
      "node *", "npm *", "npx *",
      "python *", "python3 *", "pip *",
      "cargo *", "rustc *",
      "tsc *", "biome *",
      "mkdir *", "touch *", "cp *", "mv *"
    ]
  }
}
```

**Встроенные блокировки:** Критически опасные команды (`rm -rf /`, `dd of=/dev/...`, fork-bomb и т.д.) заблокированы всегда, независимо от правил.

### mcp -- MCP-инструменты

Имена инструментов (точное совпадение). Рекомендуется разрешить часто используемые инструменты поиска:

```json
{
  "mcp": {
    "allow": [
      "web_url_read",
      "fetch_content",
      "resolve-library-id",
      "query-docs"
    ]
  }
}
```

### file -- операции с файлами

Паттерны путей для операций записи/удаления файлов:

```json
{
  "file": {
    "deny": ["/etc/*", "/usr/*"],
    "ask": ["*.config.*"]
  }
}
```

## Управление через CLI

Команда `/permissions` в интерактивном режиме позволяет просматривать и редактировать правила.

## Рекомендуемый конфиг

```json
{
  "defaultPolicy": "ask",
  "bash": {
    "allow": [
      "cat *", "ls", "ls *", "pwd",
      "echo *", "which *",
      "find *", "grep *", "rg *", "fd *",
      "head *", "tail *", "wc *",
      "sort *", "uniq *", "diff *"
    ],
    "ask": [
      "rm *", "git *",
      "node *", "npm *", "npx *",
      "python *", "python3 *",
      "mkdir *", "touch *", "cp *", "mv *"
    ]
  },
  "mcp": {
    "allow": [
      "web_url_read",
      "fetch_content",
      "resolve-library-id",
      "query-docs"
    ]
  }
}
```

Этот конфиг разрешает безопасные read-only команды, спрашивает для мутирующих операций и позволяет MCP-поиск без подтверждений.
